LeaderOS SessionCheck closes a critical security gap on LeaderOS-powered Minecraft servers: when a player changes or resets their password through the web panel, any session already active on the game server is immediately terminated. Without this plugin, a stolen account's hijacker can remain connected even after the legitimate owner regains access.

## Why You Need This

LeaderOS handles account management on the web side, but it has no built-in channel to notify the Minecraft proxy when credentials change. This means:

- A player whose account was stolen changes their password on the website.
- The attacker is **still connected** to the game server with the old session.
- The legitimate owner cannot reclaim their session until the attacker disconnects.

LeaderOS SessionCheck bridges that gap by polling a shared database table and forcibly disconnecting any affected player within seconds.

<Steps>
  <Step title="Patch LeaderOS">
    Add two small PHP snippets to `SettingsController.php` and `ResetPasswordController.php` so password events are recorded in the database.
  </Step>
  <Step title="Install the plugin">
    Drop the `.jar` into your Velocity server's `plugins/` folder.
  </Step>
  <Step title="Configure the plugin">
    Edit `plugins/leaderos-sessioncheck/config.yml` with your database credentials and preferred check interval.
  </Step>
  <Step title="Restart and verify">
    Restart the Velocity proxy. The plugin logs its status on startup, confirming the database connection and LimboAuth integration state.
  </Step>
</Steps>

<Note>
  LeaderOS SessionCheck runs entirely on your Velocity proxy and communicates with the same MySQL database used by LeaderOS. No external services, no API keys, no cloud dependencies.
</Note>
