---
title: "LimboAuth Integration: Clear Sessions After Password Change"
sidebarTitle: "LimboAuth"
description: "Enable LimboAuth integration in LeaderOS SessionCheck so players are fully forced to re-authenticate after a password change or reset from the web panel."
---

LimboAuth is a popular authentication plugin for Velocity-based offline-mode servers that holds players in a limbo state until they log in with their password. When LeaderOS SessionCheck kicks a player after a password change, you also need to clear their LimboAuth session cache — otherwise the plugin may still consider them authenticated and let them reconnect without re-entering their new password. Enabling the LimboAuth integration ensures that the kick and the session invalidation happen together, so the player is fully forced to log in again.

## How It Works

When `limboauth: true` is set in `config.yml` and the LimboAuth plugin is present on your proxy, LeaderOS SessionCheck connects to LimboAuth at startup. Each time a password reset event is detected, the plugin kicks the player from the proxy and immediately clears their LimboAuth session cache. When they reconnect they are sent back to the limbo world and must authenticate with their new password.

## Enabling LimboAuth Integration

<Steps>
  <Step title="Install LimboAuth on your Velocity proxy">
    Make sure the LimboAuth plugin JAR is present in your Velocity `plugins/` folder and that Velocity loads it successfully on startup.
  </Step>
  <Step title="Set limboauth to true in config.yml">
    Open `plugins/leaderos-sessioncheck/config.yml` and confirm the top-level `limboauth` key is set to `true`:

    ```yaml config.yml
    limboauth: true
    ```
  </Step>
  <Step title="Restart Velocity">
    Restart your Velocity proxy. Watch the console for the following confirmation line from LeaderOS SessionCheck:

    ```
    [LeaderOS-SessionCheck] LimboAuth entegrasyonu etkin.
    ```

    That message confirms the plugin found LimboAuth and is ready to clear sessions on password change.
  </Step>
</Steps>

## Disabling LimboAuth Integration

If your server does not use LimboAuth, set `limboauth` to `false`. LeaderOS SessionCheck will still kick players from the proxy when their password changes — it simply skips the session cache step.

```yaml config.yml
limboauth: false
```

## Startup Warnings

The plugin prints a status message to the console during startup to tell you whether LimboAuth integration is active. Use these messages to confirm your setup or diagnose problems.

<Tabs>
  <Tab title="Integration Active">
    ```
    [LeaderOS-SessionCheck] LimboAuth entegrasyonu etkin.
    ```

    This message means LeaderOS SessionCheck found the LimboAuth plugin and is ready to clear sessions on password change. No action needed.
  </Tab>
  <Tab title="Plugin Not Found">
    ```
    [LeaderOS-SessionCheck] limboauth: true ancak LimboAuth bulunamadı; session kaldırma devre dışı.
    ```

    This message means you set `limboauth: true` in `config.yml`, but the LimboAuth plugin was not detected on the proxy. Players will still be kicked from the proxy when their password changes, but their LimboAuth session cache will **not** be cleared. To fix this, either install LimboAuth or set `limboauth: false` if you do not intend to use it.
  </Tab>
</Tabs>

<Note>
  LimboAuth is an optional dependency. LeaderOS SessionCheck starts and operates normally with `limboauth: false`, or when LimboAuth is simply absent from the proxy — no errors are thrown in either case.
</Note>
