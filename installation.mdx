---
title: "Install LeaderOS SessionCheck on Your Velocity Server"
sidebarTitle: "Installation"
description: "Step-by-step guide to installing LeaderOS SessionCheck: create the database table, patch LeaderOS PHP controllers, and configure the Velocity plugin."
---

Installing LeaderOS SessionCheck is a two-part process. First, you prepare the LeaderOS web side by creating the shared database table and patching two PHP controllers so they record password events. Then you deploy the plugin to your Velocity server and point it at the same database. Follow every step in order — skipping the web-side patches means the plugin will never receive any events to act on.

<Steps>
  <Step title="Create the Database Table">
    Connect to the MySQL database that LeaderOS uses and run the following SQL statement. This creates the `PasswordResets` table that acts as the communication channel between the web panel and the plugin.

    ```sql
    CREATE TABLE IF NOT EXISTS `PasswordResets` (
      `id` INT(11) NOT NULL AUTO_INCREMENT,
      `username` VARCHAR(255) NOT NULL,
      `date` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP(),
      PRIMARY KEY(`id`)
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
    ```

    <Note>
      This table must be created inside the same database that LeaderOS reads and writes. If you use a separate database for the Velocity plugin, the plugin will find an empty table and never act on any password events.
    </Note>
  </Step>

  <Step title="Patch SettingsController.php">
    Open `apps/main/app/controllers/Account/SettingsController.php` inside your LeaderOS installation directory. Locate the password-change handler and add the following two lines at lines 330–331:

    ```php
    $insertPasswordReset = db()->prepare("INSERT INTO PasswordResets (username, date) VALUES (?, ?)");
    $insertPasswordReset->execute(array(auth()->user()->username(), datetime()));
    ```

    This code fires when a logged-in player submits the change-password form in the web panel. It inserts their Minecraft username and the current timestamp into the `PasswordResets` table, which the plugin will pick up on its next polling cycle.
  </Step>

  <Step title="Patch ResetPasswordController.php">
    Open `apps/main/app/controllers/Auth/ResetPasswordController.php` inside your LeaderOS installation directory. Locate the password-reset handler and add the following lines at lines 59–64:

    ```php
    $account = db()->prepare("SELECT username FROM Accounts WHERE id = ?");
    $account->execute(array($resetRequest["accountID"]));
    $username = $account->fetchColumn();

    $insertPasswordReset = db()->prepare("INSERT INTO PasswordResets (username, date) VALUES (?, ?)");
    $insertPasswordReset->execute(array($username, datetime()));
    ```

    This code fires when a player resets their password through the forgot-password flow. Because the reset is initiated without an active login session, the code first looks up the Minecraft username from the `Accounts` table using the account ID stored in the reset request, and then inserts it into `PasswordResets`.
  </Step>

  <Step title="Download and Install the Plugin">
    Download the latest `LeaderOS-SessionCheck-x.x.x.jar` from the [GitHub Releases page](https://github.com/Apcu01/LeaderOS-SessionCheck/releases). Place the downloaded `.jar` file directly into your Velocity server's `plugins/` directory.

    <Tip>
      Always use the latest release to ensure you have the most recent bug fixes and compatibility updates. Check the release notes for any breaking changes between versions.
    </Tip>
  </Step>

  <Step title="Configure the Plugin">
    Start your Velocity server once so the plugin generates its default configuration file, then stop the server again. Open `plugins/leaderos-sessioncheck/config.yml` and fill in your database credentials and preferences:

    ```yaml
    database:
      host: "localhost"
      port: 3306
      name: "leaderos"
      username: "root"
      password: ""
      pool-size: 3
      connection-timeout: 10000
      use-ssl: false

    settings:
      check-interval-seconds: 60
      debug: false

    limboauth: true

    messages:
      kick-message: "<red>Şifreniz değiştirildi. Lütfen yeni şifrenizle tekrar giriş yapın.</red>"
    ```

    Use the same `host`, `port`, `name`, `username`, and `password` values that LeaderOS uses to connect to MySQL — both sides must point at the same database. Set `limboauth: true` if you have the LimboAuth plugin installed, or `limboauth: false` if you do not. Adjust `check-interval-seconds` to control how frequently the table is polled; the recommended value is `60`.

    For full details on every configuration option, see the [Configuration](/configuration) page.
  </Step>

  <Step title="Restart the Server">
    Start your Velocity server. Watch the console for the plugin's startup confirmation message:

    ```
    [LeaderOS-SessionCheck] LeaderOS-SessionCheck etkin. Kontrol aralığı: 60sn | LimboAuth: true
    ```

    This message confirms that the plugin loaded successfully, established a database connection, and scheduled its polling task. If you see an error instead — such as a database connection failure — the plugin disables itself safely without affecting the rest of the proxy, and the full stack trace is printed to the console to help you diagnose the issue.

    <Warning>
      If the plugin logs a database connection error on startup, double-check that the credentials in `config.yml` exactly match those used by LeaderOS, that the MySQL server is reachable from the machine running Velocity, and that the `PasswordResets` table was created in step 1.
    </Warning>
  </Step>
</Steps>

<Note>
  The plugin will not interfere with normal gameplay — it only acts when a username appears in the `PasswordResets` table. Players who have not changed or reset their password are completely unaffected by the polling cycle.
</Note>
