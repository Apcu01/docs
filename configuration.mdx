On first start, LeaderOS SessionCheck automatically creates `plugins/leaderos-sessioncheck/config.yml` by copying the bundled default. You need to edit this file to match your server's database credentials and preferences, then restart Velocity for your changes to take effect.

## Default Configuration

```yaml config.yml
database:
  host: "localhost"
  port: 3306
  name: "leaderos"
  username: "root"
  password: ""
  pool-size: 3
  connection-timeout: 10000
  use-ssl: false

settings:
  check-interval-seconds: 60
  debug: false

limboauth: true

messages:
  kick-message: "<red>Şifreniz değiştirildi. Lütfen yeni şifrenizle tekrar giriş yapın.</red>"
```

## Database Connection

These settings tell the plugin how to connect to the MySQL database where LeaderOS stores its data.

<ParamField path="database.host" type="string" default="localhost">
  The hostname or IP address of your MySQL server. Use `localhost` if the database runs on the same machine as your proxy.
</ParamField>

<ParamField path="database.port" type="number" default="3306">
  The port your MySQL server listens on. The standard MySQL port is `3306`; change this only if your host uses a custom port.
</ParamField>

<ParamField path="database.name" type="string" default="leaderos">
  The name of the database where LeaderOS stores its data. This value must match the database name configured in your LeaderOS web installation.
</ParamField>

<ParamField path="database.username" type="string" default="root">
  The MySQL user the plugin connects as. This user must have read and write access to the `database.name` database.
</ParamField>

<ParamField path="database.password" type="string" default='""'>
  The password for the MySQL user. Leave this as an empty string if your MySQL user has no password set.
</ParamField>

<ParamField path="database.pool-size" type="number" default="3">
  The number of database connections kept open by the connection pool. A value of `3` is sufficient for most servers; increase it only if you observe connection contention under heavy load.
</ParamField>

<ParamField path="database.connection-timeout" type="number" default="10000">
  How many milliseconds the plugin waits before considering a connection attempt timed out. `10000` (10 seconds) is a safe default for most setups.
</ParamField>

<ParamField path="database.use-ssl" type="boolean" default="false">
  Whether to enable SSL/TLS for the MySQL connection. Set to `true` if your database host requires encrypted connections (common with managed cloud databases).
</ParamField>

## Plugin Settings

<ParamField path="settings.check-interval-seconds" type="number" default="60">
  How often, in seconds, the plugin queries the `PasswordResets` table for new password change events. The recommended value is `60`. The minimum accepted value is `1` — the plugin silently enforces this floor.
</ParamField>

<ParamField path="settings.debug" type="boolean" default="false">
  When set to `true`, the plugin logs additional diagnostic information to the console — for example, a message each time it clears a LimboAuth session. This is useful when troubleshooting connection or kick issues. Disable it in production to keep your logs clean.
</ParamField>

## LimboAuth Toggle

<ParamField path="limboauth" type="boolean" default="true">
  Whether to clear a player's LimboAuth session cache after kicking them. Set this to `true` if your proxy uses the LimboAuth plugin, so the player's authentication session is fully invalidated and they must log in again on reconnect. Set it to `false` if your server does not use LimboAuth. See the [LimboAuth Integration](/limboauth-integration) page for full details.
</ParamField>

## Messages

<ParamField path="messages.kick-message" type="string" default="<red>Şifreniz değiştirildi. Lütfen yeni şifrenizle tekrar giriş yapın.</red>">
  The message displayed to a player when they are kicked because their password was changed or reset. This field supports [MiniMessage](https://docs.advntr.dev/minimessage/format.html) formatting tags such as `<red>`, `<bold>`, and `<gold>`.

  The default value is a Turkish message meaning: *"Your password was changed. Please log in again with your new password."*
</ParamField>

<Tip>
  You can use any MiniMessage formatting in the kick message. Here are a couple of examples in English:

  ```
  <red>Your password was changed. Please log in with your new password.</red>
  <gold><bold>Security Alert:</bold></gold> <white>Password changed. Please reconnect.</white>
  ```
</Tip>

<Warning>
  The `database.name`, `database.username`, and `database.password` values must exactly match the credentials used by your LeaderOS web installation. If they point to a different database or user, the plugin will not be able to read password reset events and no players will be kicked.
</Warning>
